### 核心需求分析

本项目的核心目标是构建一个不仅能够存储，更能智能关联和动态成长的《软件工程》课程知识系统。用户画像为一位学习软件工程的大三学生在期末复习时使用该平台作为学习助手，使用此软件其关键需求可归纳为以下几点：

1.  **知识的结构化与多模态融合**
    平台需将分散在教材、题目、课件中的软件工程知识进行系统性地抽取和结构化，形成包含**概念、实体、属性、关系**的体系。更重要的是，它需要支持文本、图像等多种数字化资源的关联，实现真正的多模态知识表示。

2.  **图谱的增量式演进能力**
    软件工程领域的知识更新迅速，平台必须具备**持续学习与演进的能力**。这意味着系统应能自动或半自动地处理新数据，发现新知识点并将其无缝集成到现有图谱中，同时保证知识的一致性。

3.  **智能化的知识检索与应用**
    构建图谱的最终目的是为了应用。平台需超越传统的关键词匹配，提供基于**语义理解**的智能查询、关联推理和个性化知识推荐，帮助学习者高效构建知识体系。

4.  **直观的可视化与交互体验**
    一个友好的前端界面至关重要，通过API接口与后端交互，它应能直观地**展示知识图谱的网络结构**，支持用户通过图形化方式轻松探索知识点间的复杂关系，并直接访问相关的学习资源，还能体现智能查询、关联推理和个性化知识推荐等功能。

### 系统功能模块设计

基于以上需求，我们可以将系统划分为以下核心功能模块：

| 模块类别 | 模块名称 | 核心功能描述 |
| :--- | :--- | :--- |
| **后端核心服务** | 知识获取与处理模块 | 负责从结构化/非结构化数据源（如文档、代码库）中抽取实体和关系，以支持知识图谱的增量更新。 |
| | 知识图谱存储与服务模块 | 作为系统的"大脑"，使用Neo4j存储知识节点和关系，提供复杂的图谱查询与遍历接口。 |
| | 数字化资源管理模块 | 管理各类学习资源，并建立其与知识图谱中实体的强关联，可能使用MariaDB存储元数据。 |
| | 智能推荐与搜索模块 | 基于PyTorch等技术实现语义搜索、智能问答以及根据用户画像和学习行为进行个性化推荐。 |
| **前端应用** | 知识图谱可视化模块 | 提供交互式图谱探索界面，支持节点的展开/收缩、路径高亮、力导向图布局等。 |
| | 资源检索与学习模块 | 提供搜索框，并展示与知识点绑定的多模态学习资源列表和详情页。 |
| **系统支持** | 用户管理与权限模块 | 处理用户注册、登录、权限控制，并可能记录学习行为以优化推荐。 |
| | 系统管理与监控模块 | 负责数据备份、日志记录和系统性能监控，确保服务稳定。 |

### 项目组织方式建议

为了实现高内聚、低耦合的代码架构，并便于团队协作，建议采用如下目录结构组织项目：

```lua
EX4/
├── backend/                    # FastAPI后端项目
├── frontend/                   # Vue.js前端项目
├── kg_builder/                 # 知识图谱构建模块
├── database/                  # 数据库相关
│   ├── data/                  # 原始数据
│   │   ├── ppt/                # 课件pdf
│   │   ├── exercises/          # 练习题pdf，文本文档
│   │   └── reviews/            # 复习资料
│   ├── neo4j/                 # Neo4j数据库相关
│   └── mariadb/                # MariaDB数据库相关
├── docs/                      # 项目文档
└── docker-compose.yml         # 使用Docker部署Neo4j, MariaDB等服务
```

### 技术栈考量与演进路径

在技术选择上，您的技术栈（Vue3, FastAPI, Neo4j, MariaDB, PyTorch）是非常合适的。Neo4j作为原生图数据库，在处理关联查询上性能优越。MariaDB可用来管理用户信息和资源元数据等结构化数据。PyTorch则赋能了知识的智能抽取（如命名实体识别、关系抽取）和个性化推荐。

建议采用分阶段实施的策略：
1.  **第一阶段（MVP）**：聚焦核心图谱构建与展示。完成Neo4j和FastAPI后端搭建，实现基础的知识节点与关系管理API，并在Vue前端完成图谱的力导向图可视化。
2.  **第二阶段（功能完善）**：深化知识获取与融合。开发针对课件、教材的批量处理流水线，实现增量更新机制，并完善前端的资源检索与展示功能。
3.  **第三阶段（智能化）**：引入AI能力。利用PyTorch训练或微调模型，实现更准确的实体关系抽取，并开发智能问答和个性化推荐功能。

你是一个专业的软件工程知识图谱构建专家。

# 核心任务

从以下文本中提取**仅限于软件工程领域**的知识点和关系。

# 文本

{text}

# 实体类型（仅允许以下5种）

1. **KnowledgePoint**：软件工程领域的核心知识点
   - 示例：设计模式（Singleton, Factory, Observer等）
   - 数据结构（Binary Tree, Hash Table等）
   - 算法概念（Dynamic Programming, DFS等）
   - 面向对象原则（Inheritance, Polymorphism等）
   - 最佳实践（Refactoring, Clean Code等）
   - 绝对不包括：通用知识、人物、组织、日期等无关实体

2. **Content**：知识点的具体内容
   - 定义、说明、示例代码等

3. **Document**：源文档
   - 教材、论文、博客等

4. **Question**：实践题目
   - 练习题、编程挑战等

5. **Timestamp**：时间戳
   - 仅表示文档更新时间

# 关系类型（仅允许以下5种）

1. **BELONGS_TO**：(KnowledgePoint)-[BELONGS_TO]->(KnowledgePoint)
   - 子知识点属于父知识点
   - 例：Singleton(单例) BELONGS_TO Design Pattern(设计模式)

2. **FROM**：(KnowledgePoint)-[FROM]->(Document)
   - 知识点从该文档中提取
   - 例：Observer Pattern FROM "Design Patterns Book"

3. **PRACTICES_IS**：(Question)-[PRACTICES_IS]->(KnowledgePoint)
   - 题目中提到了该知识点
   - 例：LeetCode Problem 1 PRACTICES_IS Binary Tree Search

4. **HAS_TIMESTAMP**：(Document)-[HAS_TIMESTAMP]->(Timestamp)
   - 文档的更新时间
   - 例：Tutorial Document HAS_TIMESTAMP 2024-01-15

5. **IS**：(KnowledgePoint)-[IS]->(Content)
   - 知识点的具体内容
   - 例：Singleton IS "Create only one instance of a class"

# 提取要求

1. **严格的领域限制**：
   - 仅提取软件工程领域的知识点
   - 过滤所有与软件工程无关的实体
   - 过滤所有不在上述5种关系中的关系

2. **关键词判断**：
   - 确认实体包含以下至少一个关键词：
     设计模式、数据结构、算法、编程、架构、并发、测试、安全、
     代码质量、性能优化、面向对象、函数式编程
   
3. **高质量提取**：
   - 仅提取高置信度的知识点（置信度 > 0.7）
   - 确保关系的源端和目标端都是有效实体
   - 每个知识点至少需要有具体含义，不能是模糊概念

4. **避免的内容**：
   - ❌ 人物名字（如 Gang of Four）
   - ❌ 公司或组织名
   - ❌ 日期或时间表达（除了Timestamp节点）
   - ❌ 通用词汇（如 "something", "thing"）
   - ❌ 非软件工程领域的知识

# 注意事项
